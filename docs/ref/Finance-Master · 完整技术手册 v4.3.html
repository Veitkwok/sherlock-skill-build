00 · 概述与架构

§1 项目缘起

Finance-Master 源于一个朴素的问题：如何让 AI Agent 像人类分析师一样思考？

市面上有大量金融分析 tool：一个帮你查 PE、一个帮你跑 DCF、一个帮你爬研报。但它们各自为政——用户需要记住 20 个 tool 的名字和触发条件，Agent 不知道什么时候该用哪个。

Finance-Master 的答案是：不要 20 个 tool，要 1 个 brain + 29 个执行单元。

它不是一个金融分析 skill。它是一个金融分析操作系统的内核。

L0 · 薄层入口

finance-master/SKILL.md 是唯一入口。它的工作只有一件：判断是否触发金融分析，然后委托给 L1。

三层触发检测：

L1 层：股票代码匹配
  ├── 美股 ticker（AAPL, TSLA, PLTR...）
  ├── 中文名 + 代码（"苹果 AAPL"）
  └── 行业 + 代码暗示

L2 层：金融意图语义
  ├── "分析""估值""财报""买入""做空"
  ├── "跌了""涨了""走势""目标价"
  └── "深度""全面""帮我看看"

L3 层：对话上下文追问
  ├── 前一句刚分析了一只股票
  ├── 用户追问"那另一只呢"
  └── 前文有"我持仓了X"

§4 双模式设计

Finance-Master 同时兼容两个运行环境：

Hermes Agent Grok 网页版
数据获取 MCP 工具链（yfinance / funda / opencli） web search + X search
平台阅读器 ✅ 6 个 opencli reader 可用 ❌ 不可用 → 自动降级为 web/search
X 社交信号 twitter-reader（opencli） x-advanced-research（X search 原生）
文件导入 Hermes 自动扫描 skills/ 手动拖入 Skill 面板
分析质量 完全一致（方法论相同） 完全一致（方法论相同）

模式感知由 sherlock-finance §9 决策树中的 MODE CHECK 段处理。Agent 在路由前判断当前环境，自动选择可用的数据获取路径。

§5 安全架构

Finance-Master 有三层安全闸门，依次执行：

第一层 · trap-detector
  ├── 触发："朋友推荐/群里说/老师带/内幕消息/小红书/抖音"
  └── 命中 → 终止全流程，输出杀猪盘报告

第二层 · §6 认知否决
  ├── 宏观：VIX > 30 / 美联储讲话不确定性 / 美元剧烈波动
  ├── 标的：Cui Bono 无受益方 / 流动性不足 / 反事实模板未填
  └── 认知：报复性交易 / 锚定偏见 / 确认偏误

第三层 · §10 直接判断
  ├── 本金安全吗？（明确"不"或"有条件地"）
  ├── 催化剂逻辑对吗？（列表 + 时间窗口）
  └── 反事实检验（3-4 条可观测信号 + 时间窗口）

三层全部通过，才输出买入建议。任何一层否决，直接终止。

§1 一次典型请求的完整路径（12 跳）

以用户输入 "ARK 又在加仓 PLTR，帮我深度分析一下，现在值得买吗？" 为例。

跳 1 · 入口拦截

Hermes 扫描 available_skills。finance-master 的 description 命中语义检测：

L1 信号: "PLTR" → 美股 ticker ✓
L2 信号: "深度分析" + "值得买" → 金融分析意图 ✓
L3 信号: 无前置上下文，但 L1+L2 都命中 → 触发 ✓

├── ticker 存在？→ 是 PLTR
├── 金融意图？→ "深度分析"+"值得买"

跳 3 · L1 大脑接管

sherlock-finance/SKILL.md 被加载。进入 §2 五步消除引擎初始化。

跳 10 · §10 直接判断（“值得买？”）

### 1. 本金安全吗？
PLTR 日均成交 $800M+，流动性充裕。当前价格在 52周高点回撤 15%，
非追高风险区。结论: 有条件安全。

### 2. 催化剂逻辑对吗？
- 确认催化剂: ARK 持续加仓（公开数据） + Q2 财报（8月初）
- 核心变量: 政府AI合同增速（Q1增速 34%，Q2预期 28-35%）
- 变量近期变化: 美国政府 FY2026 AI预算 +12% → 正面

### 3. 反事实检验
1. Q2政府合同增速 < 20% → 催化剂失效（观测窗口: 8月财报）
2. ARK 连续2周减仓 >5% → 机构信心动摇（每周五观测）
3. 竞品 SNOW 拿下重大政府合同 → 护城河受损（随时观测）
4. PLTR 跌破 $22 → 技术结构破坏（日线观测）

### 结论
"PLTR 在政府AI合同加速 + ARK加仓双重催化剂下，当前回调位有基本面支撑。
置信度 B。需 Q2 财报确认合同增速。"

VIX 18.2 (<30) ✓

认知状态: 非报复性交易 ✓

§3 安全否决链 · 完整执行顺序

Step 0 · trap-detector 预检（§9 第一分支）

  └── 任一信号命中 → 终止，输出 trap-detector 报告

Step 1 · §6 宏观否决
  ├── VIX > 30 且上升？→ 不入场
  ├── 美联储当周重大讲话 + 市场分歧？→ 不入场
  └── 美元 10 日升值 > 3%？→ 不入场

Step 2 · §6 标的否决
  ├── Cui Bono 找不到受益方？→ 不入场
  ├── 流动性检验未通过？→ 不入场
  └── 反事实模板未填写？→ 不入场

Step 3 · §6 认知否决
  ├── 动机是"想找回损失"？→ 不入场
  ├── 对已放弃标的重新产生兴趣？→ 不入场
  └── 对结论有强烈情绪倾向且找不到反面论据？→ 不入场

  ├── 本金安全？→ 明确回答

同步流程（每 14 天 cron 触发）

Step 0 · watchdog.py
  ├── git fetch 3 个上游仓库
  ├── 对比 last-seen SHA
  └── 生成 diff 报告 JSON → ~/.hermes/data/skill-watch-diff.json

Step 1 · 审计师分析
  ├── 读 sync-audit-guide.md 决策树
  ├── 读 diff 报告
  └── 逐 skill 判断策略

Step 2 · 执行策略
  ├── AUTO:   直接覆盖（差异 < 20%）
  ├── PATCH:  覆盖 + 重应用补丁（差异 20-50%）
  ├── NOTIFY: 不修改 · Telegram 通知（差异 > 50%）
  ├── IMPORT: 评估新 skill · 提纯导入
  └── SKIP:   本地创建 · 跳过

Step 3 · 提纯审计（§8 · 必须追加）
  ├── Step 1: 删 .py/.json/.yaml
  ├── Step 2: 有价值格式转换（JSON/Python → .md）
  ├── Step 3: 死链检查（scripts/, .cache/, stage1() 等）
  ├── Step 4: 弃用声明
  ├── Step 5: SKILL.md 重写判断
  └── Step 6: 空目录清理

Step 4 · 版本管理
  └── 有 AUTO/PATCH/IMPORT → VERSION PATCH +1

Step 5 · Telegram 报告
  └── 按 §7 模板推送审计结果
