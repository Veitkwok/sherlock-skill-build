# Task 4 · Synthesis (v4.5)

> **v4.5:** Agent synthesis only — no `generate_synthesis()` scripts, no `dimensions.json` pipeline.  
> Inputs: dim notes D0–D17 + panel votes. Output: Stage 4 artifacts for the Markdown report + RETURN_BLOCK.

## Steps

### 4.1 Score sense-check (optional)
If you assigned 1–10 dim scores, list weakest dims as risk seeds. Scoring rules in `task2-dimension-scoring.md` are **optional heuristics** — map old 0–22 ids mentally to **D0–D19**.

### 4.2 Great Divide
Pick highest-conviction bull vs bear from panel (or frameworks). Three rounds, each with **numbers**:

```text
R1 bull / bear
R2 bull / bear
R3 bull / bear
judge_note: who won on evidence, not vibes
punchline: 20–40 chars, conflict + data (no 游资)
```

**Language ban:** 基本面良好 / 前景广阔 / 值得关注  
**Must:** digits, tension words (但是 / 代价 / 问题是)

Example punchlines (US):
- "ROE >15% for five years — still 40% above DCF base."
- "Growth panel loves it; Graham screen fails hard."

### 4.3 Risks (≥5)
Each risk: concrete metric, date, customer, regulation, or dilution — not "competition may increase."

### 4.4 Buy zones (four schools)

| Key | Logic |
|-----|--------|
| value | ≈ 0.85 × DCF fair (if DCF valid) |
| growth | growth-adjusted PE / PEG band |
| technical | MA60 / Stage-2 pivot from D2 |
| momentum | break of prior high + volume (US tape — **not** 龙虎榜) |

Every price needs a one-line formula. Currency **USD**.

### 4.5 Valuation triangle
Display DCF vs Comps vs LBO side by side; **highlight conflicts** (do not average into one fake fair value).

### 4.6 Counterfactuals
3–4 observable signals + time windows for Brain §10 / user monitoring.
