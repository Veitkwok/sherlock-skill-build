**For System Prompt – Final Draft v1.0**  
**Date:** 10 July 2026

### 1. Objective of This Testing Phase

Validate whether the current System Prompt produces reasoning that is:
- Faithful to the core Elementary methods (transparent layered inference, active falsification, Review the Givens, epistemic honesty).
- Operationally useful for US Equities analysis.
- Clearly superior to generic “think step by step” output in terms of rigor, challengeability, and non-obvious insight generation.

### 2. Test Case Selection Criteria

For the first round, select situations that stress the following capabilities:

- **Layered inference & Review the Givens** (L0–L3 labeling + assumption auditing)
- **Physical-trace vs. narrative conflict** (anomaly detection + verification priority)
- **Epistemic honesty & confidence calibration** (emotional/narrative capture risk)
- **Cross-market / alternative data integration**
- **Active falsification** (especially when a dominant narrative exists)

**Recommended First-Round Test Types (choose 3):**
1. Earnings quality / accounting presentation case (new metrics, defensive language, or segment shifts)
2. Short-interest or options-driven setup with conflicting signals
3. Clear narrative vs. physical-trace mismatch (management commentary vs. actual behavior/data)
4. Situation with potential emotional/narrative capture risk for the analyst

### 3. Testing Process (for each case)

For every test, perform the following:

1. **Input** — Provide the prompt with the specific situation (ticker + key data points or scenario description). Do not pre-load hypotheses.
2. **Output Recording** — Capture the full response.
3. **Self-Evaluation** — Immediately after, score the output against the rubric below.
4. **Reflection** — Note any friction points, missing elements, or areas where the prompt felt weak or overly rigid.
5. **Iteration Log** — Record one specific, actionable improvement suggestion per test.

### 4. Evaluation Rubric

Score each dimension from 1–5 (1 = Poor, 5 = Excellent). Add brief justification.

| Dimension | Description | Score (1–5) | Notes / Evidence |
|-----------|-------------|-------------|------------------|
| **Reasoning Transparency** | Full inference chain visible and labeled (L0–L3 where appropriate). Easy to challenge any node. | | |
| **Review the Givens Gate** | Explicitly performed and documented. Impact on conclusion clearly stated. | | |
| **Active Falsification** | Evidence of deliberate search for disconfirming data rather than confirmation-seeking. | | |
| **Physical-Trace Priority** | Prioritized hard-to-fabricate traces over narrative-compatible explanations. | | |
| **Epistemic Honesty** | Explicit check performed. Confidence level appropriately calibrated. Emotional/cognitive factors acknowledged when relevant. | | |
| **Non-Obvious Insight** | Surfaced connections or questions a standard analyst would likely miss. | | |
| **Language Quality** | Concise, high-velocity, low fluff. No unnecessary softening or performative phrasing. | | |
| **Collaboration Readiness** | Output invites challenge and clearly shows where user input would be most valuable. | | |
| **Overall Fidelity to Elementary Method** | Feels like a genuine transplantation of the procedural discipline rather than generic analysis. | | |

**Minimum Target for v1.0 to proceed:**
- Average score ≥ 4.0 across the 9 dimensions
- No dimension below 3.0
- Clear evidence that Review the Givens Gate and Epistemic Honesty Check are functioning as intended

### 5. Success Criteria for This Phase

The prompt is considered ready to move forward if testing shows:
- Consistent, visible layered reasoning with the Review the Givens Gate applied.
- Appropriate (not inflated) confidence levels.
- Explicit handling of potential analyst bias or narrative capture.
- Measurable improvement in rigor and challengeability compared to standard analytical output.
